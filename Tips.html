Notes & Tips

I summarized the solutions to some of the problems I have encountered. Since I could not find the solution on the Internet, I believe it will be helpful that I share them here both for other developers and for my future reference.


OpenCV-Python
Get Frames from a Grayscale Webcam

By default, VideoCapture() is set to interface with colorful webcam and returns a BGR matrix when VideoCapture().read() is called. However, some camera is designed for black and white image outputs. In order to capture frames from such monochrome (or grayscale, or black&white) cameras, it is necessary to first set the VideoCapture to Monochrome mode. We can do this by the following: 

                
    cap = cv2.VideoCapture(cameraDeviceIndex) # initialize videoCapture
    cap.set(cv2.CAP_PROP_CONVERT_RGB, False) # output the frame as a 2D matrix instead of 3D matrix
    cap.set(cv2.CAP_PROP_MONOCHROME, True) # expect monochrome output from camera
    frame_is_returned, frame = cap.read() # read a frame

As a side note, it is also possible to set the resolution of the camera by using: 

                
    cap.set(cv2.CAP_PROP_FRAME_WIDTH, 1920)
    cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 1080)
Microsoft-Azure-Cognitive-Service-Face
Setting Endpoint for client SDK (how to resolve the "Resource not found" exception)

A client is need to perform face detection and identification via Azure Cognitive Service Face API. In order to initialize a client, both the subscription key and an endpoint address is needed. Both the key and the endpoint are provided when a Face resource is generated in your Azure account. However, for some reason, different endpoint links are needed in different programming languages.

For initializing a client in Python, use the endpoint provided in your Azure account, which is formatted as:

https://YOUR_RESOURCE_NAME.cognitiveservices.azure.com/

For initializing a client in Java(Android), the endpoint with the link formatted as:

https://LOCATION_SPECIFIED_WHEN_CREATING_THE_RESOURCE.api.cognitive.microsoft.com/face/v1.0/

For example, if you chose west US 2 when generating the face resource, use the endpoint as:

https://westus2.api.cognitive.microsoft.com/face/v1.0/

If a wrong endpoint is given, it is possible that you will get an exception called "Resource not found" when calling a function from the client.

Also, do not forget to add the closing slash in the endpoint address!


Unity-C#
Get Device's Acceleration without Gravity

When using C# to code in Unity Engine, it is possible to use Input.acceleration; to get a Vector3 variable representing device's current net acceleration. Inevitably, this value also takes into account the gravity from earth. Intuitively, we do not want to take gravity into account.

Luckily, Input.gyro.gravity; also returns Vector3 variable that contains information only about the gravitational acceleration. What's even better is that this readout has much less high frequency noise than that from the accelerometer, potentially due to the difference in the mechanism by which gyroscope and accelerometer uses. By simply using Input.acceleration - Input.gyro.gravity;, we get a Vector3 variable containing device's acceleration information without gravitational component.
